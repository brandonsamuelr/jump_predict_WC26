# R32 true-P routing — report

Artifacts: routing table [data/submission_sheets/2026-06-28_r32_routing_table.csv](../data/submission_sheets/2026-06-28_r32_routing_table.csv)
(generated by `odds_lib/r32_routing.py`); helpers `odds_lib/time_window.py`, `odds_lib/advance_market.py`;
guards `tests/test_r32_routing.py`, `tests/test_time_window.py`. 60/60 rows routed, every provenance
field populated, **all 60 submit**.

## Gating pre-checks
- **B1 hydration-break gate: PASS.** Mandatory in all 2026 matches at ~22'/67', ~1–3' jitter
  (ESPN / World Soccer Talk / Reuters). `time_window.BREAKS_VERIFIED=True`; the helper refuses to
  price if the flag is unset (tested). Time-window rows are well-defined, not skip-class.
- **B2 advance gate: not in our feed, but markets exist externally.** 0 to-qualify keys in the Odds
  API feed — but **feed-absence ≠ market-absence**: Kalshi & Polymarket both run R32 markets.
  **Settlement read-back is mandatory** before using one: Polymarket's "advance to *knockout stages*"
  = group→R32 qualification, which is **NOT** the SportsPredict "advance to *R16*" (= win the R32
  tie). At lock: find the per-tie "to reach R16 / win-the-tie" market, verify it's ET+pens-inclusive,
  advancement-not-match-win, and not a stale outright; **de-vig both sides** (a vig-inclusive
  hand-entered number can be worse than a clean derivation); then enter as exact
  (`market_source=external_to_qualify_manual`). Else derive `P(win90)+P(draw90)·P(win|draw)` with an
  **odds-derived** `P(win|draw)`. **Flat 0.5 is prohibited as a default** (`price_advance` rejects
  `strength=0`, flags no-strength rows, allows 0.5 only on a genuinely symmetric win-edge) — tested.

## 1. Strong row classes (high / solid-medium)
- **MARKET_EXACT (12):** BTTS, regulation tie, match totals (3+ / 2-or-fewer), team totals (Brazil 2+),
  win-in-regulation, win-by-2 (handicap), ahead-at-HT (1H result), 1H 2+ goals (1H totals). Direct
  two-sided de-vig of the identical event — highest confidence.
- **ENGINE_FOUNDED goals (medium):** 2H>1H, scores-both-halves, score-first — one market-anchored
  joint goal distribution.
- **PLAYER_PROP_TIERED_DEVIG (8, medium):** scorer / SOT / score-or-assist via the tiered de-vig
  (exact two-sided where Yes&No exist; one-sided SOT → global ~1.045, **not** legacy 1.06).
- **ADVANCE_MARKET (2, medium):** exact external to-qualify market if found + settlement-verified +
  de-vigged at lock (higher value — replaces the whole derivation); else ET+pens derivation with an
  **odds-derived** `P(win|draw)`, never flat 0.5.
- **MARKET_COMPONENT_TRANSFORM (4, medium):** corner totals/comparison from the corners market.
- **TIME_WINDOW_MARKET_COMPONENT (4, medium):** goal/corner-before/after-break — volume from a
  market, × measured timing share, posterior mean.

## 2. Lower-confidence but submit-worthy
- **ENGINE_FOUNDED counts (low):** team/total SOT (concave map — no SOT market anywhere), cards
  counts/both-carded/more-cards (cards model), total offsides (EB model). Honest model reads ≥ crowd.
- **LOW_CONFIDENCE_PRIOR (10):** penalty / red / penalty-or-red, any-player brace, any-player 2+SOT,
  substitute-scores, 22+ total shots. Measured population base rates — neutral quality, still
  submit-worthy under relative-Brier (volume). Never a silent 0.50.
- **TIME_WINDOW_MODEL_COMPONENT (8, low):** offside-before-break, card-after-break(+ET),
  1H/2H-stoppage goal, sub-before-HT, card-in-first-half. Doubly-derived (model volume × timing
  share); the **only** rows eligible to become skip-candidates — but only if a directional bias is
  flagged. None is flagged today → all submit.

## 3. Skip candidates
**None in the live 60.** The skip rule keys on *positive reason to be worse than the crowd*
(unsupported route / unclear settlement / flagged directional bias), **not** on uncertainty. No
live row qualifies. The skip path exists and is tested (`submit_decision`: a flagged
`TIME_WINDOW_MODEL_COMPONENT` → skip; unflagged → submit).

## 4. Unvalidated assumptions (watch)
- **Time-window timing-shares** are parameterized (central share + uncertainty), not yet fit from
  measured minute-CDFs per event. Posterior-mean integration moderates them honestly, but the share
  *means* should be replaced with measured shares when available (refreshable).
- **SOT counts**: concave-map level + Poisson tail; tail dispersion is an open shadow question.
- **Cards model** is not market-anchored (no card market in feed).
- **Advance `P(win|draw)`** tilt (ET+pens) is a bounded, odds-derived assumption, not measured.
- **2+ SOT line** (Brobbey) may be absent/one-sided → model fallback.
- **Provider settlement** for corners/SOT may differ from SportsPredict.

## 5. Shadow metrics to track during R32
- **Prop de-vig** (auto): `capture_pending` at lock + `harvest` at review → tiered-vs-flat Brier and
  the tier-a-vs-tier-b selection-bias audit (`odds_lib/prop_devig_shadow.py`).
- **Time-window rows**: realized-vs-shipped by class (MARKET vs MODEL component) and by window —
  the first real test of the timing-share assumption.
- **SOT player-sum vs concave** (`odds_lib/prop_devig_shadow` sibling: `sot_playersum_shadow`) at
  high λ.
- **Advance derivation**: realized advance vs derived P(advance) on the 2 rows (+ future rounds).
- **Base-rate rows**: penalty/red/offside realized rates vs the population priors used.
